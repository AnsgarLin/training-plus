Flow gaps & fixes
Gap 1 — Missing DOM elements: startScanner() looks for #scan-video and #scan-canvas, but the current scan HTML never renders them. Camera silently fails every time.
Gap 2 — Camera not auto-started: User taps "Use camera" but the view re-renders the same two-button layout (no video element in DOM), so startScanner() exits immediately on if(!video||!canvas)return.
Gap 3 — Upload requires manual decode tap: After picking an image, the user still has to press "Scan QR from this image" — this step adds friction with no benefit.
Fix 1 — Render video + canvas in scan HTML: Add <video id="scan-video"> (styled as camera area) and a hidden <canvas id="scan-canvas"> directly in the scan template so startScanner() always finds them.
Fix 2 — Auto-start camera on scan view: Call startScanner() immediately when entering importView='scan' instead of waiting for a "Use camera" tap. Remove the two-button choice entirely.
Fix 3 — Auto-decode on image pick: After file input fires and importImageSrc is set, immediately call decodeQRFromImage() — no "Scan QR" button needed.
